# 投资日报系统 - 飞书版改造说明

## 📋 改造概览

本次改造将投资日报系统从**企业微信+腾讯文档**迁移到**飞书体系**，实现了消息推送、文档存储、数据归档的全面飞书化。

---

## 🔄 主要改动

### 1. 新增模块

#### `feishu_client.py` - 飞书API封装模块
提供三个核心客户端类：

- **FeishuClient**: 飞书基础API客户端
  - 消息发送（文本/富文本）
  - 文档创建
  - 文件/图片上传

- **FeishuBitableClient**: 多维表格专用客户端
  - 表格信息查询
  - 数据导出/插入/同步
  - 持仓记录、行情数据、资金动向、投资建议的专项操作

- **便捷函数**:
  - `send_feishu_message()`: 发送飞书消息
  - `write_to_feishu_doc()`: 创建飞书云文档
  - `save_to_feishu_bitable()`: 保存数据到多维表格

### 2. 主程序改造 (`main.py`)

#### 导入部分
```python
# 新增飞书模块导入
from feishu_client import (
    FeishuClient, 
    FeishuBitableClient,
    send_feishu_message,
    write_to_feishu_doc,
    save_to_feishu_bitable
)
```

#### `job_morning()` 函数
- **早盘8:00推送逻辑**：
  - 优先使用飞书推送（如果启用）
  - 回退到企业微信（如果飞书未启用）

#### `job_evening()` 函数
- **收盘15:30推送逻辑**：
  1. **简报推送**：飞书优先，企业微信备选
  2. **详报文档**：
     - 飞书云文档（如果启用）
     - 腾讯文档（备选）
  3. **文档链接推送**：推送到飞书/企业微信
  4. **多维表格存储**：
     - 保存持仓记录
     - 保存每日行情
     - 保存资金动向
     - 保存投资建议

#### 新增辅助函数
- `prepare_bitable_data()`: 准备多维表格数据
  - 提取持仓记录
  - 提取指数行情
  - 提取资金动向
  - 提取投资建议
- `extract_section()`: 从AI分析文本中提取章节内容

### 3. 配置文件更新 (`config_template.yaml`)

#### 新增飞书配置段
```yaml
feishu:
  enabled: true
  app_id: ""         # 飞书应用 App ID
  app_secret: ""     # 飞书应用 App Secret
  push_open_id: ""   # 用户 open_id（单聊推送）
  chat_id: ""        # 群聊 ID（群聊推送）

bitable:
  url: ""            # 多维表格 URL
```

#### 保留企业微信和腾讯文档配置（作为备选）
- `wecom.enabled`: 默认 `false`
- `tencent_docs.enabled`: 默认 `false`
- `smartsheet.enabled`: 默认 `false`（已禁用，改用飞书多维表格）

## 🚀 配置方法

### 步骤1：创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 创建企业自建应用
3. 获取 **App ID** 和 **App Secret**

### 步骤2：配置应用权限

在飞书应用的「权限管理」中添加以下权限：

#### 消息权限
- `im:message` - 获取与发送单聊、群组消息
- `im:message:send_as_bot` - 以应用身份发消息

#### 文档权限
- `docx:document` - 文档基础操作
- `docx:document:readonly` - 读取文档
- `docx:document` - 创建、编辑文档

#### 多维表格权限
- `bitable:app` - 多维表格基础操作
- `bitable:app:readonly` - 读取多维表格
- `bitable:record` - 读写记录

### 步骤3：获取推送目标ID

#### 单聊推送（推荐）
1. 在飞书中打开「我的」→「设置」→「基本信息」
2. 复制「用户 ID」（格式：`ou_xxxxxxx`）
3. 填入 `feishu.push_open_id`

#### 群聊推送
1. 创建飞书群聊
2. 将机器人添加到群聊
3. 获取群聊ID（格式：`oc_xxxxxxx`）
4. 填入 `feishu.chat_id`

### 步骤4：准备多维表格

1. 创建飞书多维表格，包含以下数据表：
   - **持仓记录**：股票名称、日期、持仓数、成本价、现价、盈亏金额、盈亏比例
   - **每日行情**：指数名称、日期、当前点位、涨跌幅
   - **资金动向**：资金类型、日期、净流入、状态
   - **每日投资建议**：日期、大盘观点、仓位建议等

2. 复制多维表格URL，填入 `bitable.url`

### 步骤5：填写配置文件

```bash
# 复制模板
cp config_template.yaml config.yaml

# 编辑配置文件
vim config.yaml
```

填写以下关键配置：
```yaml
feishu:
  enabled: true
  app_id: "cli_a9c0e6053c3adbc0"
  app_secret: "你的App Secret"
  push_open_id: "ou_e4b27077cea975d9b1a1cf7305de1e33"

bitable:
  url: "https://ocnvqlj4rir4.feishu.cn/base/QxhTbLTqoa0xPusz3aFc2Fuanph"
```

## 📊 数据流向

### 早盘流程 (08:00)
```
数据获取 → 简报生成 → 飞书推送
```

### 收盘流程 (15:30)
```
数据获取 → 简报生成 → 飞书推送
         ↓
    详报生成 → 飞书云文档 → 文档链接推送
         ↓
    数据提取 → 飞书多维表格
```

## 🗂️ 多维表格数据结构

### 持仓记录表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 股票名称 | text | 股票/ETF名称 |
| 日期 | datetime | 记录日期 |
| 持仓数 | number | 持仓股数 |
| 成本价 | number | 成本价 |
| 现价 | number | 当前价格 |
| 盈亏金额 | number | 盈亏金额 |
| 盈亏比例 | number | 盈亏比例(%) |

### 每日行情表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 指数名称 | text | 指数名称 |
| 日期 | datetime | 记录日期 |
| 当前点位 | number | 指数点位 |
| 涨跌幅 | number | 涨跌幅(%) |

### 资金动向表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 资金类型 | text | 资金类型（如北向资金） |
| 日期 | datetime | 记录日期 |
| 净流入 | number | 净流入金额（亿元） |
| 状态 | single_select | 流入/流出 |

### 每日投资建议表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 日期 | datetime | 记录日期 |
| 大盘观点 | single_select | 看多/看空/中性 |
| 仓位建议 | single_select | 仓位建议 |
| 建议买入 | text | 建议买入标的 |
| 建议卖出 | text | 建议卖出标的 |
| 重点关注 | text | 重点关注标的 |
| 风险提示 | text | 风险提示内容 |

## 📝 注意事项

### 1. 权限配置
确保飞书应用具有以下权限：
- 消息发送权限
- 文档创建权限
- 多维表格读写权限

### 2. 敏感信息保护
- 不要将 `config.yaml` 提交到版本控制
- 建议使用环境变量存储敏感信息

### 3. 回退机制
- 如果飞书推送失败，会自动回退到企业微信
- 如果飞书文档创建失败，会回退到腾讯文档
- 腾讯智能表格功能已禁用，改用飞书多维表格

### 4. 数据保留
- 原有的数据获取逻辑完全保留
- AI分析逻辑完全保留
- 简报生成逻辑完全保留
